# ack-documentdb-provider

Use this chart when the deployment needs an ACK-managed Amazon DocumentDB
cluster plus a connection secret that can be consumed by workloads in-cluster.

## What It Helps With

- defining the `DBCluster` resource
- exporting connection data into a secret
- optional reflector integration for cross-namespace secret use
- optional `PushSecret` integration for external secret publishing

This chart fits well when an application deployment in `argo-apps` needs the
database lifecycle to be part of the same Argo workflow.

The full values reference is generated by `helm-docs` into the chart
`README.md` during pre-commit and is included here in Backstage as the local
[Generated Reference](reference.md) page.
