# ack-opensearch-provider

Use this chart when the deployment needs an ACK-managed OpenSearch domain and a
connection secret that other workloads can consume.

## What It Helps With

- provisioning the OpenSearch `Domain` resource
- exporting endpoint and auth data into a secret
- optional reflector integration for cross-namespace secret use
- optional `PushSecret` integration for external secret publishing

This chart is commonly paired with an application `ApplicationSet` when search
infrastructure should be managed as part of the same deployment.

The full values reference is generated by `helm-docs` into the chart
`README.md` during pre-commit and is included here in Backstage as the local
[Generated Reference](reference.md) page.
